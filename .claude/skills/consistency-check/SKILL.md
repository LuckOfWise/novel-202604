# 小説プロジェクト矛盾チェック

このスキルは、小説プロジェクトの設定ファイル・プロット・本文の間の矛盾を検出し報告する。

## 実行手順

### 1. ファイルの読み込み

以下のファイルを全て読み込む：

- `CLAUDE.md`（執筆ガイドライン）
- `settings/world.md`（世界観設定）
- `settings/characters/*.md`（全キャラクター設定）
- `outline.md`（プロット・伏線管理表）
- `chapters/*.md`（全話の本文）

### 2. チェック項目

以下の7つの観点で矛盾を検出する：

#### A. キャラクター設定の整合性
- 年齢・経歴が設定ファイルと本文で一致しているか
- キャラクターの能力・制約が設定通りに描写されているか
- キャラクター間の関係性に矛盾がないか

#### B. 魔法システムの一貫性
- 改変能力のルール（素材が必要、解析の壁、暴走リスクなしなど）が一貫しているか
- 魔力の制約（自前では生活魔法のみ等）が守られているか
- 戦闘シーンでの魔法描写がworld.mdのルールに従っているか

#### C. プロットと本文の整合性
- outline.mdの各話あらすじと実際の本文が合致しているか
- プロットにある要素が本文で欠落していないか
- 本文にプロットにない大きな展開が追加されていないか

#### D. 伏線の設置と回収
- outline.mdの伏線管理表と本文を照合
- 設置されるべき話数で伏線が設置されているか
- 回収されるべき話数で伏線が回収されているか

#### E. キャラクター口調の一貫性
- 各キャラクターの口調が設定ファイルの記述通りか
- 話全体を通じて口調がブレていないか
- 特徴的な言い回し（カイの「っすよ」、ミラの「あんた」等）が維持されているか

#### F. 時間軸の一貫性
- 日数・移動距離に矛盾がないか
- 「翌日」「二日後」等の時間表現が前後の記述と整合しているか
- 移動にかかる時間が距離と矛盾していないか

#### G. CLAUDE.mdルールへの準拠
- 固有名詞の最小化ルールが守られているか（フルネーム不使用、勢力名の統一等）
- 三人称限定視点（ヴァン視点）が維持されているか
- 内心描写が十分にあるか
- セリフと地の文のバランスが取れているか

### 3. 結果の報告

検出された矛盾を以下の形式で日本語で報告する：

```
## 矛盾チェック結果

### 検出された矛盾

#### [重大な矛盾]（ストーリーに影響するもの）
- **箇所**: ファイル名と該当部分
- **内容**: 具体的な矛盾の説明
- **修正案**: 修正の方向性

#### [軽微な不整合]（許容範囲だが改善可能なもの）
- **箇所**: ファイル名と該当部分
- **内容**: 具体的な不整合の説明

### チェック済み項目サマリー
- キャラクター設定: ✓ / ✗
- 魔法システム: ✓ / ✗
- プロット整合性: ✓ / ✗
- 伏線管理: ✓ / ✗
- 口調一貫性: ✓ / ✗
- 時間軸: ✓ / ✗
- ガイドライン準拠: ✓ / ✗
```

矛盾が見つからなかった場合は「矛盾なし」と報告し、チェック済み項目のサマリーのみ表示する。

## 注意点

- 推測や解釈で矛盾を「作り出さない」。明確な矛盾のみ報告する
- 「〜くらい」等の曖昧表現は、設定値との厳密な不一致として報告しない
- 修正案は方向性のみ示し、具体的な文章の書き換えは行わない（ユーザーが判断する）
